# seo-guildford.uk brain

Static Astro 5 + Tailwind 4 brochure site, ~18 pages, no pSEO (areas/services/industries are single hub pages, not per-suburb pages). GSC property `sc-domain:seo-guildford.uk`. Repo: gh:sunnyp81/seo-guildford, deploy via Cloudflare Pages git integration (push to master auto-builds, `npm run build`, output `dist`).

## 2026-07-07 recovery pass
- Diagnosis: young-domain ramp, not a demotion. 28d GSC: impressions 1363 to 3382 (up), clicks 1 to 2. Query-level avg positions are mostly 40 to 90+ across ~45 non-branded queries (seo agency guildford, seo surrey, technical seo guildford, etc) despite real page-1-quality content, so this reads as "still earning trust/authority", not lost rankings. GSC indexing check on 10 core URLs: only homepage indexed, the other 9 are "Discovered - currently not indexed" or "URL is unknown to Google" (services/, areas/, industries/, seo-audit, seo-pricing, free-seo-audit, ai-search-optimisation, about all not indexed). This is the real blocker: Google has not chosen to crawl/index most of the site yet.
- Fixed a hard-rule violation: swept 222 em/en dash characters across 18 .astro files (pages + Base.astro layout) with a byte-safe sed pass, replacing " — " with ", " and en dashes with hyphens. Also corrected resulting comma-separated `<title>` tags to use a colon instead (e.g. "SEO Guildford: Local Search Marketing for Surrey") for cleaner SERP titles. This was a compliance fix (no-em-dash rule), not a CTR rewrite pass.
- Fixed a fabricated-stats trust liability: homepage hero stats bar (`src/pages/index.astro`) claimed "3.2x Avg Traffic Growth", "97% Client Retention", "150+ Keywords Page 1", "8 yrs Search Experience" with zero backing data, on a domain that is itself unranked. Replaced with non-numeric service pillars (Technical / Local / AI Search / Free Audit) so nothing unverifiable is asserted.
- Case study pages (`case-studies/*.astro`) already carry an explicit "this is illustrative, anonymised" disclosure both on-page and in FAQ schema, so left as-is; the fictional 5-star Review schema on the property-firm case study is a residual soft risk (fake-review-schema pattern) not fixed this session, worth a look later.
- Did not touch service depth, schema, or FAQ content: pages already have LocalBusiness/Service/FAQPage/Breadcrumb schema, quick-answer definitional blocks, and 1500+ word depth per page. The bottleneck right now is indexation/trust, not content depth.
- Build verified clean (18 pages), pushed to GitHub for CF Pages auto-deploy. Submitted changed core URLs to Bing (see Bing Webmaster). No Google "request indexing" tool available via MCP; relies on CF Pages resubmitting sitemap and organic recrawl.
- Next step for Sunny: if pages are still "Discovered - not indexed" in 2 to 3 weeks, that points to a domain-trust/backlink issue rather than a content issue, consider building a handful of genuine local citations/backlinks (Surrey Chamber, local directories) mentioned in the services page copy, since the site has none yet.
